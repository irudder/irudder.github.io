现在的思路就是：

总结webrtc里面的工程难点，然后到对应的github上去找开源项目，然后一个一个项目的研究，然后再回过头去研究工程.

1.音视频的网络抖动缓冲策略

2.网络的拥塞处理策略

3.丢包重传策略
